# GitHub Sponsors and Funding

## FUNDING.yml
- Location: `.github/FUNDING.yml`
- Adds a "Sponsor" button to the repository page
- Supports multiple funding platforms

## Supported Platforms
| Platform | Key | Example |
|----------|-----|---------|
| GitHub Sponsors | `github` | `github: username` |
| Patreon | `patreon` | `patreon: username` |
| Open Collective | `open_collective` | `open_collective: project` |
| Ko-fi | `ko_fi` | `ko_fi: username` |
| Custom | `custom` | `custom: ["https://link.com"]` |

## GitHub Sponsors
- Built-in to GitHub — no external platform needed
- Supports one-time and recurring sponsorships
- GitHub does not charge a fee (payment processing fees apply)
- Sponsors can be individuals or organizations
- Available for individual developers and organizations

## Key Exam Points
- FUNDING.yml enables the Sponsor button on repositories
- GitHub Sponsors is GitHub's built-in funding platform
- Multiple funding platforms can be listed simultaneously
- The file must be in the `.github` directory
