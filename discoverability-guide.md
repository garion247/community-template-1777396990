# GitHub Discoverability Guide

## Topics
- Tags that categorize your repository
- Appear on the repository page below the description
- Searchable on GitHub (e.g., `topic:javascript`)
- Help GitHub recommend your repo to interested users
- **Best practice**: Add 5-10 relevant topics

### Good Topic Examples
- Language: `javascript`, `python`, `go`
- Framework: `react`, `django`, `express`
- Purpose: `cli`, `library`, `api`, `template`
- Domain: `machine-learning`, `web-development`, `devops`
- Community: `hacktoberfest`, `good-first-issue`

## Repository Description
- The short text shown in search results
- Keep it concise (1-2 sentences)
- Include key functionality and purpose
- Set with: `gh repo edit --description "Your description"`

## GitHub Explore
- URL: https://github.com/explore
- Curated collections of repositories by topic
- Trending repositories (daily, weekly, monthly)
- Topics to explore by category
- Collections curated by the GitHub community

## GitHub Trending
- URL: https://github.com/trending
- Shows repositories gaining the most stars
- Filterable by language and time period
- Developers page shows trending contributors
- Great way to discover popular new projects

## Making Your Repo Discoverable
1. Write a clear, concise description
2. Add relevant topics (5-10)
3. Create a professional README with badges
4. Add a LICENSE file (required for open-source)
5. Include a CONTRIBUTING guide
6. Use issue labels like `good first issue` and `help wanted`
7. Participate in Hacktoberfest (October)

## Key Exam Points
- Topics are the primary mechanism for categorizing repos
- GitHub Explore surfaces curated and trending repositories
- The `good first issue` label helps attract new contributors
- A LICENSE file is essential for any open-source project
- Repository descriptions appear in search results
